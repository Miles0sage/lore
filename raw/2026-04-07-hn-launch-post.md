# Lore: We Audited 10 Agent Frameworks. All 10 Are Missing Cost Guards.

Two months ago, an agent we deployed ran overnight with no budget constraints. It hit a rate-limited upstream API, got caught in a retry loop, and burned $2,000 in six hours. Zero tasks completed. The agent framework we used was fine. We just didn't have the other half of the stack.

That's why we built `lore`.

## The Audit

We cloned 10 of the most popular open-source agent frameworks and ran static analysis on each one, looking for four production reliability patterns:

- **Circuit breaker**: stops cascading failures when a downstream service degrades
- **Dead letter queue**: captures and classifies failed tasks so nothing gets silently lost
- **Cost guard**: hard stop on API spend or action count
- **Observability**: structured logging and traces so operators can understand what happened

The projects: OpenAI Swarm, LangGraph, LlamaIndex, CrewAI, Pydantic AI, AutoGen, Phidata/Agno, BabyAGI, GPT-Researcher, AutoGPT.

Here's what we found:

| Pattern | Missing |
|---------|---------|
| Circuit breaker | 8/10 |
| Dead letter queue | 9/10 |
| **Cost guard** | **10/10** |
| Observability | 5/10 |

**Every single framework has zero cost control.** Not undocumented. Not partial. Zero.

## The Real Finding: CrewAI

CrewAI is one of the most production-deployed frameworks right now. 25,000+ stars. We audited it in detail. Here's what matters:

**CrewAI tracks tokens. It does not stop spending them.**

```python
# What exists: observational token counting
self.token_cost_process.sum_prompt_tokens(usage.prompt_tokens)

# What's missing: enforcement
if total_tokens > budget:
    raise CostGuardExceeded("budget exhausted")
```

Token counter is in the codebase. It increments accurately. But there's no hard stop. A 10-agent crew running unattended overnight will count every token it spends, then bill you for all of them.

We also found a critical safety gate failure: HITL webhook URLs are not persisted on resume. Your human approval gate silently stops working after the first pause. You don't know until something goes wrong.

## Why This Matters

These frameworks are genuinely excellent at capability: getting LLMs to plan, execute multi-step tasks, use tools, and delegate. That's hard. The teams behind these projects have made the ecosystem dramatically more capable.

But "can an agent do X?" and "will an agent do X safely when things go wrong at 2am?" are different questions. The second one has no dominant framework yet. The patterns exist — they're borrowed from distributed systems (circuit breakers, DLQ) — but they haven't transferred to the AI agent ecosystem.

We're proposing that they should.

## The Fix (Three Lines of Code)

Lore ships production-ready scaffolds for all four patterns. Drop them into any framework:

```bash
pip install lore-agents
lore scaffold cost_guard
lore scaffold circuit_breaker
lore scaffold dead_letter_queue
```

Each scaffold generates ~100 lines of runnable code. Takes 60 seconds to integrate.

**Cost guard on a CrewAI crew:**

```python
from lore import CostGuard

guard = CostGuard(budget_tokens=500_000, warn_at=0.80)

class GuardedCrew(Crew):
    def _run_agent_step(self, agent, task):
        result = super()._run_agent_step(agent, task)
        guard.consume(tokens=result.token_usage.total)
        return result
```

At 80% budget: log warning. At 100%: raise exception. Crew stops before the bill gets out of hand.

**Circuit breaker on a tool call:**

```python
from lore import CircuitBreaker

breaker = CircuitBreaker("external_api")

class ProtectedTool(BaseTool):
    def _run(self, query: str) -> str:
        with breaker:
            return self.external_api.call(query)
```

If the tool fails 5 times in 60 seconds, circuit opens. Next 10 calls fail instantly instead of hammering a dead endpoint.

## Run This Yourself

We built `lore audit` — a tool that scans your own codebase and tells you exactly which patterns you're missing:

```bash
lore audit . --output audit_report.json
```

Returns structured JSON. Each finding maps to a `lore scaffold` command. Most teams remediate all critical gaps in one day.

Try it on any agent project. Your own or ours. Public repo:

```bash
git clone https://github.com/Miles0sage/lore-agents /tmp/lore
lore audit /tmp/lore
```

The audit runs locally. No data leaves your machine.

## What's Still Early

This is v1.0. A few honest gaps:

- Audit findings are generated by Gemini 2.5 Pro — they're not 100% precise yet. Expect false positives on observability checks.
- We've tested the scaffolds on LangGraph, CrewAI, and vanilla Python agents. Less coverage for newer frameworks.
- The knowledge base (78 articles, 15 archetypes) is built from first principles — not yet validated against large production datasets.
- We need real user feedback on what gaps matter most. What patterns are you actually hitting in production?

What we're confident about: circuit breakers work, DLQs work, and cost guards prevent the $2,000 wake-up call. These are solved problems. The agent ecosystem just hasn't adopted them yet.

## Why Open Source

Cost guards and circuit breakers are not a competitive advantage. If one team has them and another doesn't, the playing field is just broken. We're open-sourcing all scaffolds under MIT because the ecosystem needs the reliability layer more than we need proprietary moat.

Enterprise will want: audit CI gates, compliance reporting, fleet management. That's where we'll build sustainable business. But the core patterns should be free and portable.
